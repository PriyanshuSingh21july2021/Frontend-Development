Rules for Identifiers in JavaScript

Allowed Characters:

Identifiers can consist of letters (both uppercase and lowercase), digits (0-9), underscores (_), and dollar signs ($). Identifiers cannot start with a digit.

  • ✅ Valid: myVar, _value, $price
  • ❌ Invalid: 123abc, 9name

Case Sensitivity:

JavaScript is case-sensitive, meaning myVar and myvar are considered two different identifiers.

  • ✅ Valid: MyVariable, myVariable
  • ❌ Invalid: Misuse of case if the identifiers are meant to be the same.

Keywords and Reserved Words:

Identifiers cannot be JavaScript reserved words or keywords (such as var, let, function, class, if, while, etc.).

  • ✅ Valid: myFunction, variableName
  • ❌ Invalid: function, if, while

Unicode Support:

JavaScript identifiers can contain Unicode characters (including non-Latin letters), which allows the use of various language scripts.

  • ✅ Valid: π, 变量, élément
  • ❌ Invalid: Use of symbols like !, @, #, etc.

No Spaces:

Identifiers cannot contain spaces.

  • ✅ Valid: userName, totalAmount
  • ❌ Invalid: user name, total amount

Starting with $ or _:

Identifiers can start with a dollar sign ($) or an underscore (_), often used in naming conventions for frameworks or private variables.

  • ✅ Valid: $myVar, _privateVar
  • ❌ Invalid: var123 (starts with a number)

Examples:


// Valid identifiers
let name = "Priyanshu";
let $price = 100;
let _counter = 1;
let π = 3.14;  // Using a Greek letter as an identifier

// Invalid identifiers
let 123abc;  // Cannot start with a digit
let var;     // 'var' is a reserved keyword
let user name;  // Spaces are not allowed

Naming Conventions (Best Practices):

  • CamelCase for variables and functions: myVariable, myFunction
  • PascalCase for class names: MyClass
  • Uppercase letters for constants: MAX_VALUE

Following these rules ensures that your JavaScript code is valid and readable.
